The changes we have made:

index.html

The image isn’t as big as in the design pages because the original pictures were too small to be stretched like that. The font is also a bit bigger to fill out the space.

news.html

We removed the text since we thought that most people will understand what a facebook-feed is. The feed also gets more focus now.

Products

We thought that there was too many categories in products so we shortened down the list a bit. A few of the categories was pretty similar, so it would basically be giving the same information several times. Some of the categories was also too narrow to deserve a dedicated page. Therefore the dropdown-menu also contains fewer elements.

about.html

We changed the text into something more relevant and removed one picture so the focus would be more on the persons. We also changed the heading into something more thought provoking.

contact.html

Minor changes of the boldness of the headings to make them more “heading-like”.

resize_navbar_when_scrolling.js

We changed this one entirely since we realized you don’t really scroll a lot at our site. Instead we added a show_time.js on the shop-page so it will show if the shop is open or not.

CSS

Instead of just having one css-file (main.css) we split it up a bit since it became kind of hard to keep track of, so we now have several css-files.

Modularity

We used a main.css so we didn’t need that many different css-files. We also made the javascript-functions general, so they could be used on different places. All were in charge. Ideally we would also use a separate html-document for the navbar and link this into the others using php. This would allow us to change the navbar without having to go into each and every html to make the same change. We probably also could achieve this modularity with javascript, but php seems like the most used method for this task.
